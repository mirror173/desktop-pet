# QQ人桌宠

QQ人桌宠是一款运行在 Windows 上的 Electron 桌面宠物应用。它以“桌宠互动 + 效率工具 + 飞书能力接入”为核心，提供日常陪伴、待办处理、文件搜索、时间提醒、工作梳理和部分办公场景的一键入口。

## 项目特色

- 桌宠互动：支持角色对话、状态反馈和日常陪伴式交互
- 效率工具：集成文件搜索、计划处理、时间格式化、文本处理等常用能力
- 办公接入：支持飞书相关能力，包括登录、消息、会议和周报等场景
- 多面板工作台：将常用能力收纳到不同面板中，方便快速切换
- 轻量可扩展：基于 Electron + CommonJS 组织，便于继续扩展和维护
- AI 与工具协同：不是单纯聊天机器人，而是把 AI 能力嵌入真实工作流中，形成可落地的桌面协作入口

## 功能概览

### 1. 桌宠互动

- 点击角色触发对话和状态变化
- 支持气泡消息展示
- 支持不同输入模式下的互动反馈

### 2. 文件与任务处理

- 本地文件搜索
- 待办与计划类操作入口
- 部分任务型面板的快速处理
- 支持常见排序、筛选和打开操作

### 3. 时间与提醒

- 计划时间解析
- 时间格式化展示
- 任务提醒与时间相关提示
- 周期性任务与日程场景支持

### 4. 飞书能力接入

- 飞书登录与状态查看
- 群聊与消息相关能力
- 会议信息查询
- 周报、到岗梳理等辅助能力

### 5. 工作台面板

- 控制中心
- 轨迹/打卡类面板
- 检查与复盘相关面板
- 高级快捷入口

### 6. SmartTODO 外部任务协同

项目中集成了 `SmartTODO` 作为外部任务协同入口。点击相关功能后，桌宠会连接到 `SmartTODO` 的网页服务，用于承接待办、任务管理或外部工作流跳转。

这个设计的意义在于：

- 保持桌宠本体轻量，不把所有能力都硬编码进本地应用
- 让桌宠作为统一入口，连接到已有的任务服务生态
- 展现“本地桌宠 + 外部 SaaS 服务”的协同思路
- 方便后续把更多轻量能力通过链接或接口方式接入，而不是不断膨胀桌面端代码

## 架构说明

本项目采用“主进程 + 渲染进程 + 服务层 + 工具层”的分层方式组织：

- `src/main/`：负责窗口管理、托盘、快捷键、IPC 注册、外部服务调用
- `src/renderer/`：负责页面交互、面板逻辑、输入响应、展示层辅助能力
- `src/shared/`：负责跨进程共享的配置、路径和通用工具
- `docs/`：记录设计方案、重构思路和项目成果，方便评委快速理解整体架构

这种结构的好处是：

- 业务职责清晰，便于定位问题
- 主进程与渲染进程边界明确，降低耦合
- 新功能可以按模块继续扩展，而不是继续堆进单个入口文件
- 适合长期迭代，也更方便演示项目工程化能力
